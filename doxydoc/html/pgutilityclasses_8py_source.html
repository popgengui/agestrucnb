23 class IndependantProcessGroup( object ):
24  '''
25  Simple convenience wrapper around a set of processes,
26  allowing calls into the instance to add a process,
27  or terminate all processes. This class is motivated
28  by implementing parallell replicate AgeStructureNe-inititated
29  simuPop simulations in a PGGuiSimuPop instance via
30  python's multiprocessing class, which apparently interacts
31  with simuPOP such that the multiprocessing.Pool and
32  multiprocessing.Queue process managers do not create
33  simuPOP objects independant of each other. This
34  necessitated using separately instantiated and started
35  multiprocessing.Process objects. Because the typical
36  case will be to run y replicates using x processes, with
37  y>>x, then we need to manage the set of x processes, replacing
38  those finished with fresh processes.
39 
40  All member processes are assumed to be completely
41  independant of all others, so that any I/O messes
42  caused by calling terminate() will not lock up
43  the parent process
44  '''

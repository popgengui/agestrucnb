30  '''
31  wraps 2 dictionaries, one for header and loci byte addresses, the other for
32  byte addresses of "pop" lines and individual addresses. The header dict
33  has one level, the pop dict has 3 levels:
34  outer key: for the pop dict, pop number: 1,2..., for header/loci simply the line number
35  middle key: pop dict only, item number: 0,1,2
36  for the pop dictionsary the first "pop" line itself is the zeroth item
37  and for pop 1,2,..., the entry "pop"
38  innermost key: item-line number, 0,1,2...
39  this key needed because genepop allows multi line entries for
40  a single individual
41  value: byte address of the first byte of the line
42 
43  example, if (as most often) the individual entry takes up only one line,
44  to find the entry for the 2nd indiv at pop 2, point the file pointer at
45  the byte address given by self.__byte_addresses[2][2][1], and execute a readline().
46  If this individual was on more than one line, say 2 lines, the entry would 8
47  be given by concatenating the lines read at self.__byte_addresses[2][2][1]
48  and self.__byte_addresses[2][2][2]

50  Note that we aim to parse the genepop file in accordance with the format described

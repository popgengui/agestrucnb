27  '''
28  Class to read and write the table
29  files with Ne estimations generated
30  by pgdriveneestimator.py.
31 
32  Motivation: Needed in order to filter the
33  table for plotting using Brian T's
34  line regresss program, when the table
35  has more than one subsample value for
36  the "pop" sections and/or "loci" list. For example,
37  if the user has subsampled a genepop file
38  to get 95 and 85 percent fot individuals
39  per pop, and and also subsampled the loci
40  to get 95 and 85 percent of loci (however
41  many replicates in each), then the reaults
42  will need to be filtered to include only
43  rows whose pop subsdample value i and loci
44  subsample value j are fixed, so that i,j
45  is 85,85 or 85,95 or 95,85 or 95,95.
46 
47  The file is loaded into a numpy array,
48  and all fields are typed as bytes.
49  (There's a type problem if we type the array
50  "str", so that we get different types depending
51  on py2 vs py3:
52 
53  "if you create a NumPy array containing strings,
54  the array will use the numpy.string_ type (or
55  the numpy.unicode_ type in Python 3)." More precisely,
56  the array will use a sub-datatype of np.string_
57 
58  This class returns its items as strings, after
59  decoding the bytes using sys.stdout.encoding.
60  I think this gives items compatible with
61  the string methods in both py2 and py3.

65  I am adding code to return the data lines
66  grouped on an arbitrary number of fields
67  (with const limits on total grouping,

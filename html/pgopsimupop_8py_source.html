1037  def __get_dict_chromosome_number_by_loci_number_evenly_distributed ( self,
1038  i_number_of_loci,
1039  i_number_of_chromosomes ):
1040 
1041  '''
1042  2018_04_18. We're adding chromosome/loci
1043  associations in order to leverage LDNe2's
1044  feature that computes Ne using chromosome/loci
1045  information. Note that the easy, default way
1046  to get simupop to evenly assign, say, 30 loci
1047  to 5 chromosomes, "loci=[10,10,10],
1048  chromtypes=[Autosome]*5", means that if our user
1049  is using both q Msats and r snps, and since we always
1050  assign the first q loci to be msats and the last r
1051  to be snps, the default chrom assignments will tend to
1052  bunch the msats together on the same (few) first
1053  chromosomes and the snps on the last. We thus
1054  will distribute them using the addLoci and addChrom
1055  methods of the pop struct (see __createSinglePop).
1056  In order to hold these assignments
1057  steady among replicates, we, rather than using any
1058  random assignment, we assign N loci 0,1,2...N-1 to
1059  M chromosomes 0,1,2...M-1, in order, i=1,2,3...N,
1060  assigning loci i to chromsome i mod M, which puts
1061  an additional loci on each of the first T chromosomes
1062  where T = N mod M.
1063 
1064  Note: if the number of chromosomes exceeds the number
1065  of loci, we trim the number of chromosomes to equal
1066  the number of loci.
1067 
1068  2018_05_22. Testing using Genepop's linkage diseq
1069  test shows that despite assigning loci using the
1070  alternating-chromosome method described above, alleles
1071  are treated as physically linked according
1072  to the scheme, for N loci and M chromosomes, that the first
1073  N/M loci are assigned to a single chromosome, the second
1074  N/M loci assigned to another, etc. Therefor we change the
1075  loci assignment scheme so that the program will output

1077  are linked in the program.

1084  #end if more chromosomes thatn loci, get rid of excess
1085  #chromosomes
1086

1088

1090

1092
